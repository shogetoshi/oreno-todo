# 0054 カレンダーイベントの上書き抑止 - 実装計画書

## Issue概要

カレンダーイベントを2回以上インポートした場合、同じIDのイベントが既に存在するとき無条件で上書きしていた。
以下の条件で上書き判定を制御する。

- **上書きしない（既存データを保持）**
  - すでに `timeRanges` に値が入っている場合
  - 完了状態（`completedAt` が null でない）の場合
- **上書きする**
  - それ以外（`timeRanges` が空かつ未完了）

## 実装内容

### 1. Model層: `TodoRepository.addCalendarEventsToItems` の上書き判定ロジックを変更

**対象ファイル**: `src/models/TodoRepository.ts`

現在の `addCalendarEventsToItems` は、新しいイベントと同じIDを持つ既存アイテムをすべて除外してから新しいイベントを追加するという実装になっており、常に上書きしています。

これを以下の構造に変更します。

1. 既存アイテムをIDでマッピングした `Map` を作成する
2. 新しいイベントごとに、既存イベントが存在するかチェックする
3. 既存イベントが存在する場合、`shouldOverwriteCalendarEvent` で上書き可否を判定する
4. 上書き不可なら既存を保持し、それ以外は新しいイベントを採用する
5. 新しいイベントのIDに一致しない既存アイテムはそのまま末尾に保持する

`shouldOverwriteCalendarEvent` を `TodoRepository` の private static メソッドとして追加します。

```
shouldOverwriteCalendarEvent(existingEvent: CalendarEvent): boolean
```

- `timeRanges` に要素がある場合 → `false`（上書きしない）
- `isCompleted()` が true の場合 → `false`（上書きしない）
- それ以外 → `true`（上書きする）

既存アイテムが `CalendarEvent` 型でない場合（型不一致）は `true`（上書きする）を返します。

### 2. テスト: `TodoRepository.test.ts` に上書き抑止のテストケースを追加

**対象ファイル**: `src/models/TodoRepository.test.ts`

`addCalendarEventsToItems` の describe ブロックに以下のケースを追加します。

- `timeRanges` が空かつ未完了のイベントは上書きされる
- `timeRanges` に値が入っているイベントは上書きされない（既存を保持）
- 完了済みのイベントは上書きされない（既存を保持）
- 上書き保持と上書き対象が混在する場合に正しく動作する

## 考慮事項・注意点

- `addCalendarEventsToItems` のシグネチャは変更しません。呼び出し元（`useTodos.ts` の `importCalendarEvents`）への変更は不要です
- `shouldOverwriteCalendarEvent` はメソッドに切り出すことで、判定ロジックが独立してテスト可能になります
- イミュータブルな設計を維持します（新しい配列を返す）
- 型チェックで `CalendarEvent` 以外のアイテムと偶然IDが衝突する場合は、安全のため上書きします（通常は発生しない）
