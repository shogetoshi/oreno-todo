# Issue #0009: 完了済みTodoに対する計測開始時の挙動改善

## 概要
完了済みのTodoに対して計測開始ボタンを押した際、完了状態を解除しつつ計測を開始する挙動に変更する。

## 現状の問題
現在、完了済みTodoに対して計測開始を実行した場合の挙動が未定義。

## 要件

### 機能要件
- 完了済み（`completedAt !== null`）のTodoに対して計測開始（`startTimer()`）を実行した場合、以下の処理を行う：
  1. 完了状態を解除（`completedAt`を`null`に設定）
  2. 計測を開始（`timeRanges`に新しい`TimeRange`を追加）
  3. `updatedAt`を現在時刻に更新

### 実装方針
- `Todo.startTimer()`メソッド内で完了状態をチェックし、完了済みの場合は自動的に完了を解除する
- 新しいメソッドは作成せず、既存の`startTimer()`メソッドの挙動を変更する

### ユーザー体験の考慮
- 完了済みタスクに対して計測ボタンを押す = 「このタスクをもう一度やる」という明確な意図がある
- ワンクリックで意図した状態（未完了 + 計測中）になる
- 「計測開始 = タスク作業中」という状態の一貫性を保つ

## 実装時の注意事項
- イミュータブルな設計を維持（新しいインスタンスを返す）
- 単体テストを追加（完了済みTodoに対する`startTimer()`のテストケース）
- MVCアーキテクチャのModel層のみの変更で完結すること

## テストケース
- [ ] 完了済みTodoに対して`startTimer()`を実行すると、`completedAt`が`null`になる
- [ ] 完了済みTodoに対して`startTimer()`を実行すると、新しい`TimeRange`が追加される
- [ ] 完了済みTodoに対して`startTimer()`を実行すると、`updatedAt`が更新される
- [ ] 未完了Todoに対する`startTimer()`の挙動は変わらない（既存の挙動を維持）
